CRICKETLIVE STUDIO

Complete deployment guide — Render.com + Netlify

💚 100% Free Hosting
⚠️

Why two services? Netlify only hosts static files. Your app needs a Node.js server for real-time WebSocket (Socket.IO) signaling, video relay, and room management. Render.com hosts this for free. Both are connected via environment variables.

Before You Start

You need free accounts on:

  • GitHub.com — to store your code (free)
  • Render.com — to host the Node.js server (free)
  • Netlify.com — to host the React app (free)

You need on your computer:

  • Node.js v18+ — nodejs.org
  • Git — git-scm.com

Deployment Steps

1
Extract the zip & open terminal
Manual

Unzip cricket-live-studio-v2.zip and open a terminal inside the cricket-v2 folder.

# On Mac/Linux:
cd ~/Downloads/cricket-v2

# On Windows (PowerShell):
cd C:\Users\YourName\Downloads\cricket-v2
2
Push code to GitHub
FREE

Create a new GitHub repository, then run:

copy & paste git init
git add .
git commit -m "CricketLive Studio v2"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/cricket-live-studio.git
git push -u origin main
Tip: Go to github.com/new, create a repo named cricket-live-studio, then copy the URL it shows you.
3
Deploy server to Render.com
FREE

Sign up at render.com with GitHub. Then click New Web Service.

→ Open Render Dashboard

Use these exact settings:

FIELDVALUE
Namecricket-live-server
Repositoryyour GitHub repo
Root Directoryserver
RuntimeNode
Build Commandnpm install
Start Commandnode index.js
Instance TypeFree

After deploy completes, copy your URL:

YOUR RENDER URL https://cricket-live-server.onrender.com
4
Update the server URL in your code
1 line edit

Open client/.env.production and set your Render URL:

REACT_APP_SERVER_URL = https://cricket-live-server.onrender.com

Save the file, then push to GitHub:

git add client/.env.production
git commit -m "Set Render server URL"
git push
5
Deploy frontend to Netlify
FREE

Sign up at netlify.com with GitHub.

→ Open Netlify Dashboard

  1. Click "Add new site" → "Import an existing project"
  2. Choose GitHub → select your cricket-live-studio repo
  3. Netlify reads netlify.toml automatically — settings are pre-filled
  4. Click "Deploy site"

After deploy, copy your Netlify URL:

YOUR NETLIFY URL https://cricket-studio-abc123.netlify.app
Optional custom name: In Netlify → Site settings → Domain management → Change site name to something like mycricketlive → your URL becomes https://mycricketlive.netlify.app
6
Connect Render ↔ Netlify
1 variable

Go back to Render → your service → Environment tab → Add variable:

CLIENT_URL = https://cricket-studio-abc123.netlify.app

Render redeploys automatically. This makes QR codes and watch links point to your live Netlify URL.

7
Set up auto-deploy (optional but recommended)
Auto CI/CD

The .github/workflows/deploy.yml file is included. To enable auto-deploy on every push:

  1. Go to Netlify → Site settings → Build & deploy → Build hooks → Generate a build hook URL
  2. Go to Netlify → User settings → Personal access tokens → Generate token
  3. Go to Netlify → Site settings → General → copy Site ID
  4. In your GitHub repo → Settings → Secrets and variables → Actions → add:
    SECRET NAMEVALUE
    NETLIFY_AUTH_TOKENyour personal access token
    NETLIFY_SITE_IDyour site ID
    REACT_APP_SERVER_URLyour Render URL

Now every git push to main automatically rebuilds and deploys your app.

8
Before each match — wake up the server
30 seconds

Render free tier sleeps after 15 min of inactivity. Before a match, run:

bash wake.sh https://cricket-live-server.onrender.com

Or simply open this URL in your browser and wait for the JSON response:

HEALTH CHECK https://cricket-live-server.onrender.com/api/health
Expected response: {"status":"ok","ip":"...","rooms":0}
Takes ~30–60 seconds from cold start. After waking, it stays awake for the entire match.
